## 1. 架构概述

### 1.1 认证与授权分离

| 层级 | 职责 | 项目 |
|------|------|------|
| **认证（Authentication）** | OIDC 令牌签发、用户登录/登出、Token 刷新 | OpenFindBearings.Identity |
| **授权（Authorization）** | 系统级 Scope（admin/mobile/sync） | OpenFindBearings.Identity |
| **RBAC** | 细粒度角色-权限映射 | 各业务系统自行管理 |

OpenFindBearings.Identity 只负责 OIDC 认证和系统级 Scope，不管理各系统的细粒度 RBAC。每个业务系统（Admin、Sync、Crawler）自行维护角色-权限映射。

### 1.2 Admin RBAC 存储

Admin 项目拥有独立 PostgreSQL 数据库 `db_admin`，包含以下权限相关表：

| 表名 | 用途 |
|------|------|
| `admin_user_roles` | 用户-角色映射 |
| `admin_role_permissions` | 角色-权限映射（种子数据初始化） |
| `admin_audit_logs` | 操作审计日志 |
| `admin_configs` | 系统配置 |

业务数据通过各项目 API 代理访问，不直连业务数据库。

### 2.2 Admin 端 PermissionKey 枚举

```csharp
public enum PermissionKey
{
    DashboardView,      // 查看仪表盘
    BearingView,        // 查看商品列表
    BearingCreate,      // 创建商品（品牌/类型/轴承）
    BearingEdit,        // 编辑商品
    BearingDelete,      // 删除商品（软删除）
    DataRestore,        // 恢复已删除数据
    DataHardDelete,     // 彻底删除数据（不可恢复，超级管理员专属）
    MerchantView,       // 查看商家列表
    MerchantManage,     // 管理商家（创建/编辑/删除/恢复）
    MerchantVerify,     // 商家认证审核
    CorrectionReview,   // 信息纠错审核
    EtlManage,          // 触发/管理 ETL 任务
    CrawlerManage,      // 触发/管理爬虫
    RoleManage,         // 管理角色和权限
    UserManage,         // 管理用户角色分配
    SystemView,         // 查看系统配置
    SystemManage,       // 修改系统配置
    AuditView           // 查看审计日志
}
```

> 枚举值用于 Admin UI 显示，API 端点使用 snake_case 字符串（如 `bearing.view`）。SeedData 中的权限键使用 snake_case 与 API 对齐。

## 5. 认证集成方案（Phase 2 待实现）

### 5.1 OAuth 2.0 授权码流程

```
用户浏览器 → Admin 登录页 → Identity /connect/authorize
    → 用户登录授权
    → 回调 Admin /callback
    → Admin 用 code 换 token
    → 存储 Cookie（access_token + refresh_token）
```

### 5.2 Admin 配置项

```json
{
  "OpenIddict": {
    "Authority": "https://localhost:7201",
    "ClientId": "admin_client",
    "ClientSecret": "admin_secret",
    "Scope": ["openid", "profile", "admin"]
  }
}
```

### 5.3 权限校验流程

1. 请求进入 Admin → Cookie 提取 access_token
2. 调用 Identity /userinfo 获取用户信息和 Scope
3. 查询 `admin_user_roles` 获取用户角色
4. 查询 `admin_role_permissions` 获取角色权限
5. 检查所需权限是否在用户权限列表中

## 6. 数据库 Schema

### 6.1 admin_user_roles

```sql
CREATE TABLE admin_user_roles (
    id UUID PRIMARY KEY,
    user_id UUID NOT NULL,
    role_name VARCHAR(50) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 6.2 admin_role_permissions

```sql
CREATE TABLE admin_role_permissions (
    id UUID PRIMARY KEY,
    role_name VARCHAR(50) NOT NULL,
    permission_key VARCHAR(50) NOT NULL,
    granted BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 6.3 admin_audit_logs

```sql
CREATE TABLE admin_audit_logs (
    id UUID PRIMARY KEY,
    user_id UUID,
    action VARCHAR(50) NOT NULL,
    entity_type VARCHAR(50) NOT NULL,
    entity_id UUID,
    old_value JSONB,
    new_value JSONB,
    ip_address VARCHAR(45),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 6.4 admin_configs

```sql
CREATE TABLE admin_configs (
    id UUID PRIMARY KEY,
    config_key VARCHAR(100) NOT NULL UNIQUE,
    config_value TEXT,
    description VARCHAR(500),
    updated_by UUID,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```
